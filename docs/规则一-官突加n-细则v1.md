# 集结杯 · 双打规则一（官突 + n）细则 v1（草稿）

> 数据基线：官方 52 标准因子点数（灰机 live 核对）+ 官突池 153 条已按点数和分档。配置表见同目录 `官突ABC配置_官突池.csv` / `因子点数配置.csv`，全部可编辑、框架运行时读取。

## 0. 基线
- **平台**：CM（承载官方合作内容）；**赛事内容**：官方指挥官 + 官方因子。
- **难度**：残酷（Brutal）。
- **形式**：双打 2 人；指挥官限制与"五后二改/独狼/原神"另见《指挥官规则》。
- 规则二（纯随 12+n）已砍，不做。

## 1. 赛制结构：3 把，难度递增
- 每队打 **3 把**：第 1 把 **A（易）**、第 2 把 **B（中）**、第 3 把 **C（难）**。
- （顺序固定 or 随机 = 可调旋钮）

## 2. 每把的官突（系统随机）
- **官突池**：官方 153 条，按"因子点数和"分档（阈值可调）：
  - A 易：点数和 ≤ 8（56 条）
  - B 中：9–12（60 条）
  - C 难：≥ 13（33 条）
- **排除**：4 条限定突变（捉鸡行动 / 暴力之夜 / 灾难之轮 / 感恩季，官方进不了自定义）；外加图/因子 ban 名单（配置）。
- **抽取**：每档内**均匀随机**抽 1 条即可——难度已被档位锁死，档内随机不会失控。

## 3. 额外因子（玩家自分配）—— 这是策略层
- 选手按自身水平申报 **n**（n ≥ 1，建议封顶 ≤ 6 防爆）。
- 系统从「可用因子池」随机抽 **n 个**：
  - 池 = 官方 52 标准因子 −（双打 ban：`拿钱说话` + 恶意 bug 因子）
  - **去重**：抽到的额外因子默认不与所在把官突已含因子重复（叠加与否 = 旋钮）。
- **分配**：n 个因子由**玩家自己**分到 3 把；每把因子总数 ≤ 官方自定义上限（10）。
  - 策略点：把硬因子塞进易场拉平、还是叠到强势场，由选手决定。

## 4. 随机算法（控方差，核心）
方差大的根因是"按个数随机"，解法是"**点数预算 + 带约束采样**"：

1. **官突**：各档内 uniform 随机抽 1（难度已被档位区间锁定）。
2. **额外 n 因子**（两种模式，二选一，建议默认 A）：
   - **A. 点数预算**：设目标总点数 `T`（默认 `T ≈ n × 4`，可调）与容差 `ε`；贪心抽因子累加，逼近 `T ± ε`，因子数 = n；超了回退/替换。→ 难度落点稳定，随机的只是"哪几个因子"。
   - **B. 纯随机**：直接随 n 个（接受"随到算倒霉"的方差）。
3. **可复现**：记录随机种子，便于直播复现 / 申诉公平。

## 5. 可调旋钮（全在配置表里，框架运行时可改）
| 旋钮 | 默认 | 说明 |
|---|---|---|
| ABC 阈值 | ≤8 / 9–12 / ≥13 | 改阈值即重新分档 |
| 各档突变池 | 自动按阈值 | 可手动增删某条突变归档 |
| 图 / 因子 ban 名单 | 拿钱说话 + 限定 | 可加恶意 bug 因子 |
| n 上限 | ≤6 | 选手申报上限 |
| 额外因子模式 | 点数预算 | A 预算 / B 纯随 |
| 预算 T、容差 ε | n×4、±2 | 控方差强度 |
| 官突去重 | 开 | 额外因子是否避开官突已含 |
| ABC 顺序 | 固定递增 | 固定 or 随机 |

## 6. 待土豆拍的点
1. n 的范围 / 上限（建议 1–6）
2. 额外因子用「点数预算」还是「纯随机」
3. 额外因子能否与官突因子重复 / 叠加
4. C 档阈值取 ≥13（33 条）还是严格 ≥15（11 条）
5. ABC 三把顺序固定递增还是随机
