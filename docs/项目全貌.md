# 集结杯 · 项目全貌（v1）

> 一句话：重办「集结杯」星际2合作模式赛事。**土豆**主办(B站直播)、平台用 **CM**(自制合作 mod)、赛事内容=**官方 18 指挥官 + 官方因子 + 残酷难度**、**XP** 大框架、**yb** 抽签软件微调与 CM 侧适配。
> `jijiebei` = Cocos Creator 抽签/分档工具（产出"指挥官+因子+地图"组合）；`CM` = 选手实际开局的 SC2 mod 平台。**两者无数据对接，靠人工搬运。**

## 1. 平台关系：CM ↔ jijiebei（链路是断的，纯人工）
经双侧代码 + 文档 + 会议转录三方证据：**无任何数据对接**。
1. 选手在 jijiebei 抽到「指挥官+因子+地图」；
2. 进 CM 游戏，在**「自定义突变」UI 里手动逐个点选**这些官方因子 + 选指挥官/地图/残酷 → 开局；
3. CM 把点选写进 UserData「Mutators」`Auto-Selected/Actived`，开局逐个 dispatch 因子三段式（开启/内容/关闭，`RegisterMutator` 注册，`LibCOMU.galaxy`）。

- CM **没有**"因子码导入/一键配局"（全仓 grep `ImportMutator/MutatorCode/分享码/LoadPreset` = 0）。
- CM 原生能力**已足够**跑赛事内容（自定义突变支持任意官方因子+残酷+指挥官/地图），**不依赖任何 CM 新开发**。
- CM 在建的 launcher/跳转/公告是 CM 自身大厅体验，与抽签对接无关。

## 3. 规则体系
- **难度尺** = 官方因子点数(1-10)，**官突难度 = 其因子点数之和**。残酷+预算：+1=4-6 / +2=7-8 / +3=9-10 / +4=11-12 / +5=15-16 / +6=19-20。
- **官突池**：老 `factors.txt` 253 条(id<200=153)，按点数和分 **A≤8 / B 9-12 / C≥13**（`docs/官突ABC配置_官突池.csv`）。
- **单刷**(已实现)：8/10/12 因子三档；主流程用 `getJijieFactor` 按组概率随机。
- **双打**(待落地，只做规则一)：**官突+n** —— 每把 1 个官突(第1把A/第2把B/第3把C，自带地图+3因子) + n 个按点数预算随机的额外因子(T≈n×4±ε)，玩家自分配 3 把。规则二已砍。
- **指挥官**：S/A/B/C/O + 两将档位和≥4；**五后二改 = B 组 7 人**(凯拉克斯/诺娃/德哈卡/泰凯斯/泽拉图/斯台特曼/蒙斯克)，抽候选 4A+2B、选择 B 只能 1 个（jijie2 已有此基础，可复用）。ban 名单(`ban指挥官.txt`)=凯拉克斯/凯瑞甘。
