# Harness-Pro — jijiebei Adapter

> Generated by `harness-pro scaffold-adapter`.
> Customize reviewer config, model config, and verify checks for this repo.

## What this adapter means

This repo can run harness-pro sessions through the core CLI. The default runtime root is `.harness-pro`, with named sessions using `.harness-pro-<slug>`.

## Stage 1.8+ command shape

```bash
# Init and start
npx harness-pro init --goal "..." --phases 1 --scaffold codex --runtime-dir .harness-pro
npx harness-pro phase --phase 1 --runtime-dir .harness-pro

# Deterministic verify is an API call from harness-pro-core, not a CLI subcommand.
# Use runVerify(...) to write verify-report-N.json before review.

# Review / gate / close
npx harness-pro review --phase 1 --runtime-dir .harness-pro
npx harness-pro record-gate --phase 1 --runtime-dir .harness-pro
npx harness-pro stop --runtime-dir .harness-pro
npx harness-pro audit --runtime-dir .harness-pro
```

For a first offline smoke, add `--stub-mode` to `review`. That proves wiring and produces `bootstrap_pass`; it is not a native real-review pass.

## Reviewer config

Current reviewer count: **6**.

Use CLI for common reviewer edits:

```bash
npx harness-pro reviewer-list --config config/harness-pro-reviewers.json
npx harness-pro reviewer-set --config config/harness-pro-reviewers.json --id <reviewer> --enabled false
npx harness-pro reviewer-set --config config/harness-pro-reviewers.json --id <reviewer> --enabled false --apply
```

`reviewer-set` is dry-run by default; `--apply` writes atomically.

## Rubric evidence boundary

For real reviewer rounds, each rubric should cite pre-review deterministic evidence:

```text
evidence-bundle-pre-review-N.json#/...
```

Do not ask reviewers to judge current-round `gate-matrix-N.json`, `audit-phase-N.json`, mutation checks, host real-use, or other reviewer verdicts. Those facts exist after review and belong in audit/acceptance checks.

## Guard / enforcement

```bash
npx harness-pro guard --check pre-write --file <path>
npx harness-pro guard --check pre-bash --command "<cmd>"
npx harness-pro guard --check pre-commit --repo-root .
npx harness-pro doctor --runtime-dir .harness-pro
```

Codex uses `.Codex/settings.json` hooks. Codex/CI should enable `.githooks/pre-commit`:

```bash
sh scripts/install-harness-git-hooks.sh
```
