【6 主题 token（不变，已在 r1-theme.css）】style-{metal|sc2|minimal} × mode-{dark|light}。核心变量：--accent（金属=金#D8B15A / 星际2=绿#56cf8c / 极简=蓝#4f8ff5）、--ink、--muted、--panel-bg、--panel-edge、--drop-edge、--win/--bonus/--lose、--on-accent。所有新组件用变量，不写死颜色。三主题×明暗都要适配；正文对比度 ≥4.5:1。

【交付 5 个交互 + 规格板】每个交互给：metal-dark 主稿 + sc2-dark + minimal-light 各一屏（共用同弹层骨架），外加状态矩阵规格板（照 R1 的 SpecBoard/mx 格式）。1280×720 基准，最终在 Cocos Creator 2.4 程序化复刻（Graphics 圆角矩形+描边 / Sprite / 九宫格），所以请标注尺寸/圆角/描边宽/动效时长，不要依赖网页特效。

────────────────────────────
① BP 面板（开局第一道关，主形态：ban 1 因子 / 自选 1 指挥官 二选一）
────────────────────────────
插入位置：home 点模式 → 【BP 面板】→ 路径二选弹层（手选/随机）→ select。
当前赛制 = 二选一单选：选手要么「ban 掉 1 个不想要的因子」，要么「自选 1 个指挥官」，只能选其中一条路。框架未来可配成「都给」「都不给」「ban N」，本稿只画当前形态，但版式要能容纳两条都启用。

版式：复用 current-dialog-pathchoice 的居中弹层 + 遮罩。弹层标题「BP 阶段 · 二选一」。内放两张并排「路径卡」（比路径二选的按钮更大、带内容预览）：
- 路径卡 A「Ban 因子」：卡内嵌 ② 的揉因子横排缩略（4~5 个绿框因子，其中标 1 个被划掉=已 ban 预览）+ 一行说明「从揉出的因子里禁掉 1 个不想要的」。
- 路径卡 B「自选指挥官」：卡内嵌 1 个 CC 指挥官卡占位 + 「指定 1 名本局自选指挥官」。
两卡互斥单选：选中卡 = accent 描边高亮 + 角标 ✓（沿用 R1 ✓ 角标 pop 180ms back-out）；未选卡 = 60% dim。底部「确认 BP ▶」主按钮（沿用 R1 .startbtn 样式），未选时 disabled。
给：默认态（两卡未选）/ 选中 A 态 / 选中 B 态，三态。

────────────────────────────
② Ban 因子交互（点开路径卡 A 后的操作页）
────────────────────────────
机制：系统「多揉几个」因子（目标场数所需 + 额外 ban 数，默认揉出比正常多几个），平铺成一排 FX 绿框因子（沿用 R1 .factor-row，size≈66）。选手点 1 个标记为「ban」。
- 被 ban 的因子：整框转 --lose 红描边 + 半透灰膜（dim）+ 中心叠一个「⊘ 禁用」红角标（复用 R1 fx-clear 的 ✕ 视觉，但语义=ban，配红色 lose 而非中性灰）；旁边浮一条说明「已禁用：核弹打击」。
- 其余因子保持普通绿框。
- 单选语义：再点别的因子=改 ban 目标（旧的恢复绿框，新的转红，140ms 反向动画）。
- 顶部一行进度提示「禁掉 1 个因子（剩余可用 N）」。
- 底部「确认禁用 ▶」。
给：揉出态（全绿框未 ban）/ 已 ban 1 个态。

────────────────────────────
③ 自选指挥官交互（点开路径卡 B 后的操作页）
────────────────────────────
机制：复用 R1 select 面板「自选指挥官」分组（avatar-grid，CC 卡 60×72）的版式，但这里是独立选择页，选 1 名作为本局自选指挥官。
- 全量指挥官网格（A 组弱 + B 组强 + 拯救池排除项已灰），CC 卡可选；选中=accent 外圈 + ✓ 角标（R1 cc-sel）。
- 已 ban 的指挥官（凯拉克斯/凯瑞甘，本赛事已从 ban 名单剔除——以实际配置为准）和不可选项=ghost/dim 态。
- 顶部「自选 1 名指挥官」；选中后顶部高亮显示已选卡缩略 + 名字。
- 底部「确认自选 ▶」。
给：未选态（网格全可选）/ 选中 1 名态（顶部回显）。

────────────────────────────
④ 点金交互（赛前固定名单 + 选手每场点金，两者都要；金框=分值×2 强化）
────────────────────────────
点金 = 把某因子从普通绿框「升级」成金色强化金框（asset-border-factor-gold-grandmaster.png 样式），软件层含义=该因子难度分值×2，抽取概率不变，纯标记+翻倍。两个入口都要画：

(a) 赛前固定金框：在因子池/揉因子横排里，已在固定名单的因子直接显示金框（R1 已有 gold 态，FX 的 .gold 类）。

(b) 选手每场「点金」动作：在 select 面板每场槽位的因子上，提供「点金」手势。设计这个交互的 3 态：
  - 普通态：绿框因子 + 右上角一个小「点金」徽标（一个金色◆钻石或小锤icon，--accent 色，hover 才显，避免干扰）。
  - 点金中（hover/按下）：徽标放大 + 因子框预演金色描边渐入。
  - 已点金态：整框换成金框（金色描边 + 四角钻石铆钉，对齐 asset-border-factor-gold）；因子角标显示「×2」分值徽标（金底深字，复用 R1 锁定角标=深底金字的反相：金底+--on-accent 深字）；再点=取消点金（金框→绿框，反向 140ms）。
  - 每场点金有数量门控（默认每场可点金 1 个），点满后其余因子的点金徽标变 dim 不可点。
给：普通态（带隐藏点金徽标）/ 点金 hover 预演态 / 已点金（金框+×2 角标）态。
另给一个「点金前后对比」小规格：同一因子 绿框 vs 金框 并排 + 标注「分值 ×2 · 抽取概率不变」。

────────────────────────────
⑤ 额外难度开关（开关 + n 可配，默认 n=2）
────────────────────────────
「额外难度」是一个开关：开启后每场在 base 因子数上 +n（如 10→10+2）。放在 BP 面板里（或 home 模式卡上方的一行设置条），作为赛前设置。
设计：一个 R1 风格的 toggle 开关行——
  - 标签「额外难度」+ 副说明「每场额外 +n 个因子（更难）」。
  - 开关 toggle（off=灰轨 / on=--accent 轨 + 圆头滑块，滑动 160ms）。
  - n 的步进器：「− [ 2 ] +」（默认 2，范围 1~3 示意），accent 描边的小数字框 + 两个 ± 按钮。off 时 n 步进器 dim 不可调。
  - 开启后给一行预览反馈「本局每场因子：10 → 12」。
给：关闭态 / 开启 n=2 态 / 开启 n=3 态。

【约束（与 R1 相同）】
- 1280×720；Cocos 2.4 程序化复刻（Graphics/Sprite/九宫格），标尺寸/圆角/描边/动效时长。

- 弹层族复用 current-dialog-pathchoice 的遮罩+居中卡骨架，BP 面板与它视觉同族、可串成一条流程。
