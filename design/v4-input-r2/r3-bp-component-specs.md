## 〇、R1 设计语言摘要（项目空，内嵌供参考）

- **画布**：1280×720，Cocos 程序化复刻标尺寸/圆角/描边/动效时长。
- **6 主题 token**：`metal / sc2 / minimal` × `dark / light`，CSS 变量驱动（--gold 金色 accent ≈ #e8b84b、--ink 文字、--panel-bg/--panel-edge 面板、--muted 次要）。本规格每组件须出 **metal-dark / sc2-dark / minimal-light** 三屏。
- **复用组件**：
  - **FX（因子框）**：PNG 整图 normal/gold 双边框 + 圆环 + 可选 ✓角标 + tag（锁定/官突）；尺寸参数化（池 66 / 槽 52）。
  - **CC（指挥官卡）**：微圆角 + 内外亮光描边 + gloss + 名牌；选中/拖拽/鬼影态 + 可选 ✓角标。
  - **DropCell**：空槽虚线 + hint 文字（指挥官/因子/锁定）。
  - **CtrlBar**：顶部分段按钮（已汉化：主界面/选择/对战…）。
  - **ToastV**：校验失败浮条（!图标 + 文案 + +N 计数）。
- **选择面板布局**：`topbar(BrandLockup logo+集结杯艺术字 + 选手/模式 meta) / slots(3场:地图+锁定因子+manualSlots手选槽) / pool(因子池 + A/B/自选指挥官池) / startbtn`。
- **已实现可参考**：已选因子/指挥官打 ✓ 对勾、点金金框角标（金/非金 toggle）、随机填充按钮。

## 二、6 个 component 级需求

### C1 · BP 面板（二选一主形态）
- **触发**：该模式 BP 开关=开时，进选择面板前插入。
- **结构**：两张路径卡横排互斥单选 —— ①「Ban 因子」②「自选指挥官」。
- **状态矩阵**：未选 / 选中（accent 描边 + ✓角标 + 微缩放）/ hover / 禁用（灰掉）。
- **互斥**：单选（互斥=1），选一张另一张回未选。
- **下方**：确认进入对应分支（→ C2 或 C3）。
- 出 3 主题三屏 + 状态矩阵。

### C2 · Ban 因子
- **进入**：BP 面板选「Ban 因子」后。
- **结构**：揉横排因子（base + banN 个，默认 banN=1）—— 比正常多揉 1 个，选手点掉不想要的，剩下进手选。
- **交互**：点因子 → ban 态（划叉/暗化/移出）；ban 满 banN 个 → 确认。
- **复用**：FX 因子框；ban 态新增「已 ban」视觉（暗化+✗角标）。
- **状态**：可 ban / 已 ban / 锁定（不可 ban）。

### C3 · 自选指挥官（含无自选区模式独立路径）
- **进入**：BP 面板选「自选指挥官」后。
- **结构**：从全量指挥官池自选 1 个（替换某场随机指挥官）。
- **⚠️ 关键（O4/O5 拍板）**：拯救/非酋等 **selfShow=false（无叠加层自选区）的模式，BP 开启时也走此分支**——需一条**独立的 BP 自选 UI 路径**，与选择面板的「叠加层自选区」解耦（两套自选语义：叠加层自选区=拖拽额外指挥官；BP 自选=二选一的一条路径）。
- **复用**：CC 指挥官卡 + 自选池网格布局。
- **状态**：可选 / 已选（✓）/ 不可选（B 组占用超限等）。

### C4 · 点金（3 态 + ×2 角标）
- **位置**：选择面板因子上（池 + 已选槽），已部分实现可对齐。
- **3 态**：①未金（灰角标）②赛前固定金框（金边框，赛前名单）③运行时点金（选手每场点金，金角标高亮）。
- **角标**：金色「金」按钮 toggle + 分值 **×2** 显示标识（×2 仅视觉，不入排名）。
- **复用**：FX gold 边框 + GoldBadge（已实现，需补 ×2 标识 + 赛前/运行时两态区分）。

### C5 · 额外难度开关
- **位置**：模式配置/赛前设置区。
- **结构**：toggle（开/关）+ n 步进器（默认 2，可 +/−）。
- **说明文案**：「每场 base+n 因子，末场额外 +n」。
- **复用**：CtrlBar 风格开关 + 数字步进。

### C6 · 每模式 BP 开关配置面（今天新增重点）
- **位置**：模式选择处或赛前配置面。
- **结构**：每模式一行/一卡 + BP 开关 toggle，体现默认态：
  - 8因子 / 10因子 → 默认**开**
  - 拯救 → 默认**关**
  - 极难（=12因子）→ 默认**关**，但**可手动开**（非锁死，开关可点，区别于"灰掉不可点"）
  - 双打（官突plus / 非酋）→ 可配（本轮设计先行）
- **关键视觉**：极难「默认关但可开」要和「锁死不可」明显区分（如默认关但开关亮色可点 vs 灰掉）。
- 出 3 主题三屏 + 全模式开关态矩阵。
