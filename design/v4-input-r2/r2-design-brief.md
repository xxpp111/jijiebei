## R2 任务（三件 + 一件可选）

### 1. 对战页（battle）边框落位
现状见 `current-battle-doubles-sc2-dark.png`。把 R1 边框系统应用到对战页：
- 每场行内的因子图标 → 绿框/金框（56×56 档），官突/锁定角标沿用 R1；
- 指挥官头像（58×70 档）→ R1 指挥官卡（含名牌）；双打=并排两张卡；
- 判定三按钮（胜利/带奖励/失败）样式不变，但请核对与新边框的视觉平衡；
- 顶部控制条在 battle 页的形态（与 R1 select 页一致，直播中可收起）。

### 2. 直播浮层 OBS 底部横条（全新形态，重点）
现状全屏版见 `current-overlay-fullscreen-needs-bottombar-metal-dark.png`——它整体缩小后字不可读。
- 设计**底部横条专用形态：1280×200~240**；使用场景=OBS 浏览器源把它缩小到约 1/3 放在直播画面底部，**所有文字与图标以"缩至 1/3 仍可读"为硬标准**；
- 内容：三场（地图缩略+双指挥官+带框因子+状态徽章[胜利/进行中/待战/带奖励/失败]）+ 大比分 + 当前场次强调；
- 信息优先级：当前进行场次 > 比分 > 各场阵容；
- 6 主题 token 沿用；正文对比度 ≥4.5:1。

### 3. 二选层（手选阵容 / 随机抽签）正式化
现状兜底版见 `current-dialog-pathchoice-manual-or-random-metal-dark.png`。标准模式开局时的二选弹层：
- 两个大按钮（手选阵容=拖拽编排 / 随机抽签=一键自动分配直达对战）+ 简短说明文案；
- 遮罩、入场/出场动效、三主题×明暗适配；
- 视觉语言与 R1 控制条/按钮族一致。

### 4.（可选）双打 select 变体核对
R1 select 是单刷形态。双打（3 场 × 每场 2 指挥官槽 + 官突锁定格 ×2 + 3 额外因子槽，见 `current-battle-doubles` 的同期 select 现状）在 R1 组件体系下是否能直接成立？如有布局问题给出变体方案。

## 约束（与 R1 相同）
- 1280×720 基准（横条为 1280×H）；Cocos 2.4 程序化复刻（Graphics/Sprite/九宫格）；
- 6 主题（metal/sc2/minimal × dark/light）token 体系不变；
- 交付 = 设计画布 + 规格板（状态矩阵/尺寸/动效/Cocos 要点，照 R1 的 spec 板格式）+ 最好附 `_review.html` 式的纵排审查页便于工程逐屏截图。
