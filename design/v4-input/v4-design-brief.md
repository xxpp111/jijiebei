## 1. 因子边框系统（核心需求）

- 所有因子图标（池子 66×66/74×74、槽位 52×52、battle 56×56、result 46×46、浮层 52×52）统一加**游戏同款边框**：普通因子用 mutatorframe 绿框语义；**赛事已确认会使用 CM 金色强化因子——金色版用 grandmaster 金框区分，普通/金色两态都必须设计**。
- 边框需适配 6 主题 × 明暗（绿框在浅底的可读性处理）。
- 与现有状态系统兼容：sel（上浮+✓+accent 外圈）、dim、官突/锁定角标如何与边框叠加，请给出状态矩阵。

## 2. 指挥官边框系统

- 参考游戏内合作任务选人界面：**微圆角长方形 + 内外亮光描边 + 投影**的质感。
- 需要的状态：池中常态 / 已用（sel）/ 拖拽中 / 槽位 filled（当前是 1px accent 直角框）。
- 尺寸族：70×84（A/B 池）、66×80、60×72（自选区）、56×67（槽位）、58×70 / 64×78（battle/result/浮层）。
- 底部名字浮层（现状：黑底白字 10px）与边框的关系请一并设计。

## 3. 全局导航与控制条（新功能）

直播实操需求：软件出问题/抽取不满意时能快速回退重来。

- **屏幕切换**：顶部提供 home/select/battle/result（/浮层）的来回切换入口——与现有顶部主题切换器（3 风格 + 暗/亮，顶部中央）的关系请统一设计成一个控制条（直播时可能需要隐藏/弱化，请考虑收起态）。
- **重新随机按钮**：重抽当前局的池子（地图/因子/指挥官），停留在 select。
- **回主界面按钮**：放弃当前局回 home。
- 以上按钮的位置、形态、确认交互（误触保护：重新随机/回主界面建议二次确认）请设计。

## 4. 兜底转正清单（开发期用 token 红字/裸面板先行的）

1. **锁定因子格**：select/battle/result 中"锁定"与"官突"角标格的正式样式（建议结合 §1 边框系统：官突/锁定因子直接用游戏边框+角标）。
2. **校验错误提示**：当前为 startbtn 左侧裸红字（"第X场指挥官未选择"/"B组指挥官只能选1个"），请设计正式形态（toast/抖动/红字+图标均可）。
3. **二选层**（标准模式的"手选阵容 / 随机抽签"）：当前是兜底面板+两按钮，请按各主题语言正式化。
4. **清槽交互**：点击已填槽=清空（已实现），但无视觉提示用户"可点击清除"——请设计 hover/常驻提示与清空动效。
5. **池 sel ✓ 角标**：静态样式 v3 已定，请补拖入/清除时的过渡动效规格（时长/缓动）。

## 5. 直播浮层 OBS 底部横条形态（挂账已久的核心需求）

- 使用场景已确认：**OBS 浏览器源 + Interact 交互，单窗口**；浮层会被**缩小放在直播画面底部**。
- 当前浮层是全屏 1280×720 排版，整体缩小后字号不可读。请设计**底部横条专用形态**（建议 1280×200~240）：三场（地图缩略 + 双指挥官 + 因子带边框 + 状态徽章）+ 比分横排铺满，**字号与图标以"缩至 1/3 仍可读"为标准**。
- 信息优先级：当前进行场次 > 比分 > 各场阵容。保留 6 主题 token。

## 6. 约束（沿用 v3）

- 1280×720 基准；Cocos 2.4 程序化实现（Graphics/Sprite/九宫格），边框素材可直接出 png（请给九宫格切片建议）；6 主题 token 体系不变；OBS 对比度要求（正文 ≥4.5:1）不变。
- 设计稿仍按"由工程翻译成 Cocos"交付，重点页面：select（边框+控制条+按钮全家福）、battle、浮层横条。
