## 1) 总评

**整体达标度：结构与画板层面 70% 达标，定位框架层面集体不达标。**

六篇里 5 篇画板（总览/模式/架构/协作）都是合格甚至偏优的 Visualizer 风，层级清晰、无溢出/重叠/乱码——**基础工艺过关**。但三个系统性问题贯穿全库：

1. **定位框架全军覆没（最关键）**：6 篇里 5 篇 `framingAligned=false`。全库把软件讲成"抽签 + 记分前端"，**没有一篇在显著位置点出内核 = "按比赛规则随机生成突变因子(Mutator)组合"**。仅"当期配置"勉强对齐。定稿前必须先统一的总闸。

2. **两处硬性事实错误会误导使用者**：
   - 〔当期配置〕"8/10 因子额外删除"写错——把只在 **8 因子/拯救** 排除的 风暴英雄/虚空裂隙/30%同化体 说成 8/10 通用；真正全模式排除的是 **进攻部署**。已对 `JijieContro.ts` 行 193/214-218/233 核实，**正文写反了**。（注：〔比赛模式〕篇同样有这条错，一并改。）
   - 〔软件架构〕文件名 `JijieControl.ts` 实为 `JijieContro.ts`（结尾无 l，类名才带 l）。作为"能定位代码"的页，会让人打不开文件。

3. **面向公众普遍欠账**：突变因子/官突/降权/ban/smallRate/splice/jinanArr/A组B组 等术语多篇零解释，"开放问题"篇尤其是纯代码黑话堆砌。

## 2) 🎨 配色改进方案（全库画板统一）

### 现状问题（各篇画板目检归纳）
- **单页色阶过多**：〔比赛模式〕一行同时叠"行底色 + 特性 pill 色 + 难度星级色"三套色阶，七模式七种马卡龙色块并列。
- **个别高饱和跳色破坏统一**：〔协作流程〕"AI 工具"整行高饱和紫底，比周围重、跳。
- **装饰性噪点**：〔协作流程〕飞轮中央橙色虚线大圆无语义，还越过节点边框。
- **留白偏紧**：〔比赛模式〕底部判定 bar 几乎贴满深色底。
- **靠"花色"而非"明度"做层级**：多篇用不同色相区分区块，导致"杂"。

### 一套更克制优雅的调色方案（建议全库画板统一）

**用色原则（四条硬规则）**
1. **三色封顶**：每张画板正文区颜色相数 ≤ 主色 + 1 辅色 + 中性灰阶；状态色只在警示/标注未实现时出现。
2. **层级靠明度，不靠色相**：同一层级用同色相不同明度（深蓝标题 / 中蓝卡片 / 浅蓝底）。
3. **金色只做强调**：accent 金棕仅用于标题分隔线 + 唯一最高优先级元素，不铺面。
4. **留白优先**：节点间距、容器外边距宁松勿紧；底部深色 bar 至少留 16-24px 外边距。

**具体色板**

| 角色 | Hex | 用途 |
|---|---|---|
| 主色-深 | `#1F2D3D` | 标题、深色收口 bar、主流程节点 |
| 主色-中 | `#3E5871` | 二级卡片底、流程节点 |
| 主色-浅 | `#DCE4ED` | 卡片浅底、分组容器 |
| Accent 金棕 | `#B8893A` | 标题双分隔线、唯一强调路径（双打旁路）|
| 中性底 | `#F7F8FA` | 整页背景 |
| 中性描边 | `#C9D1DA` | 卡片细描边、泳道分隔 |
| 文字主 | `#2B2B2B` | 正文 |
| 状态-警示 | `#C0563B`（克制砖红）| "已知不一致 / 未实现 / 保留" 唯一警示色 |
| 状态-成功（可选）| `#5B8C6E`（低饱和灰绿）| 仅"已确认 / 通过"必要时 |

> 关键改法：〔比赛模式〕七色行底统一成"主色-浅同色系明暗 + 难度仅用星数"；〔协作流程〕紫底降为主色-中、删装饰虚线圆；状态全库只用 `#C0563B` 一种警示色。
